# 🚀 Epitext 프로젝트 실행 가이드

## 📋 시스템 요구사항

| 항목 | 최소 버전 | 권장 버전 |
|---|---|---|
| Python | 3.8 | 3.10+ |
| Node.js | 16.x | 18.x+ |
| CUDA | - | 11.8+ |
| RAM | 8GB | 16GB+ |

***

## Backend 실행

### 1. 가상환경 생성 및 활성화
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS/Linux
python3 -m venv venv
source venv/bin/activate
2. 패키지 설치
bash
cd backend
pip install -r requirements.txt
3. 환경변수 설정
bash
# .env.example 복사
copy .env.example .env  # Windows
cp .env.example .env    # macOS/Linux

# .env 파일 편집 (필수)
# GEMINI_API_KEY=your_api_key_here
# DATABASE_URL=sqlite:///./epitext.db
4. 서버 실행
bash
python app.py
접속: http://localhost:8000

Frontend 실행
1. 패키지 설치
bash
cd frontend
npm install
2. 환경변수 설정
bash
# .env.example 복사
copy .env.example .env  # Windows
cp .env.example .env    # macOS/Linux

# .env 파일 편집 (필수)
# REACT_APP_BACKEND_URL=http://localhost:8000
3. 개발 서버 실행
bash
npm start
접속: http://localhost:3000

Docker로 전체 실행 (권장)
1. Docker Compose 실행
bash
# 전체 빌드 및 실행
docker-compose up -d

# 로그 확인
docker-compose logs -f

# 중지
docker-compose down
2. 개별 컨테이너 확인
bash
# 실행 중인 컨테이너 확인
docker ps

# Backend 로그
docker-compose logs backend

# Frontend 로그
docker-compose logs frontend
접속:

Frontend: http://localhost:3000

Backend API: http://localhost:8000

주의사항
필수 사항
API 키 설정: Gemini API 키 없이는 번역 기능 사용 불가

포트 충돌: 3000, 8000번 포트가 이미 사용 중이면 에러 발생

방화벽: Windows Defender에서 Python/Node 허용 필요

선택 사항
GPU 사용: CUDA 11.8+ 설치 시 모델 추론 속도 향상

데이터베이스: 기본 SQLite 사용, PostgreSQL로 변경 가능

일반적인 문제 해결
bash
# 1. 포트 충돌 시
netstat -ano | findstr :8000  # Windows
lsof -i :8000                 # macOS/Linux

# 2. 패키지 설치 실패 시
pip install --upgrade pip
pip install -r requirements.txt --no-cache-dir

# 3. Node 모듈 오류 시
rm -rf node_modules package-lock.json
npm install
